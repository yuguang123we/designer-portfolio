# 设计说明：海报式创意开发者作品集

参考站点：https://wodniack.dev/

## 设计定位

这是一个面向创意开发者、交互设计师或技术型作品集的视觉方向。整体不追求传统作品集的“清爽展示”，而是把页面做成一张可滚动、可交互、带实验气质的数字海报。

核心印象：

- 极高对比度：红黑或纸白黑的强烈视觉冲突。
- 工程感网格：页面被细线、边框、分栏和编号系统严格组织。
- 字体主导：大标题是第一视觉资产，图片和案例反而退居其后。
- 代码质感：二进制、斜线、编号、短代码、微型文案构成装饰层。
- 动态背景：线场、透视网格、字符墙、滚动过场制造“正在运行”的感觉。

## 视觉关键词

- 创意开发
- 实验海报
- WebGL / Canvas
- 排版系统
- 代码仪表盘
- 复古计算机
- 印刷网格
- 高反差
- 极简但不安静

## 色彩系统

主色不需要很多，重点是足够果断。

```css
:root {
  --color-ink: #160000;
  --color-signal: #f40c3f;
  --color-paper: #fff2ed;
  --color-line: currentColor;
  --color-muted: rgba(22, 0, 0, 0.62);
}
```

主题建议：

- 默认主题：`--background: var(--color-signal)`，`--foreground: var(--color-ink)`。
- 对比主题：`--background: var(--color-paper)`，`--foreground: var(--color-ink)`。
- 深色区块：`--background: var(--color-ink)`，`--foreground: var(--color-signal)` 或 `var(--color-paper)`。

注意：这里的主题切换不是普通深浅模式，而是“海报底色模式”。切换时应保留同一套线框、边界、字体和动效逻辑。

## 字体系统

参考站点的字体角色非常清晰：

- 巨型标题字体：极窄、极重、压缩感强，适合全大写英文。
- 正文字体：带有编辑感的衬线字体，营造独立杂志和艺术档案气质。
- 辅助字体：等宽字体，用于编号、标签、导航、坐标、二进制和系统提示。

可替代方案：

```css
--font-display: "Bigger Display", "Druk Wide", "Anton", "Impact", sans-serif;
--font-serif: "Editorial New", "Noto Serif SC", "Songti SC", serif;
--font-mono: "Fraktion Mono", "IBM Plex Mono", "JetBrains Mono", monospace;
```

排版原则：

- H1 使用超大字号、紧行高、全大写，允许文字接近屏幕边缘。
- H2 可以做成小型黑底标签，使用等宽大写字母。
- 正文保持窄栏，行高舒展，文字像杂志专栏而不是产品说明。
- 导航、编号和元信息使用等宽字体，形成系统界面感。

## 页面结构

建议页面分为以下区域：

1. 固定或首屏可见的顶部控制条
   - 左侧品牌标记。
   - 中间导航：About、Work、Contact。
   - 右侧社交入口、主题切换、可用状态和联系 CTA。

2. 首屏视觉海报
   - 顶部为动态线场或透视网格。
   - 中间穿插二进制、斜线、编号等细条信息。
   - 主标题占据大面积，例如 `CREATIVE + DEVELOPER`。

3. About 区域
   - 中央窄栏正文。
   - 左右两侧保留大面积结构线、透视线或装饰网格。
   - 内容表达个人经历、能力边界、合作方式和创作态度。

4. Awards / Metrics 区域
   - 可做成黑底模块、拼贴式徽章或滚动卡片。
   - 不需要平均排布，允许遮挡、错位、裁切。

5. Work 区域
   - 使用长列表而非普通卡片墙。
   - 每个项目包含名称、短编号、序号、年份或外链状态。
   - 悬停时可触发图像预览、字符扰动、背景变化或光标反馈。

6. Contact 区域
   - 用大型文字或字符动画收尾。
   - 邮箱是主要行动点，避免复杂表单。

## 布局规则

- 页面外层保留细边框，形成“画布边界”。
- 使用 1px 线条划分区域，线条颜色跟随前景色。
- 首屏可以是 12 栏或 24 栏思维，但视觉上更像工程图纸。
- 正文最大宽度建议控制在 `520px - 680px`。
- 巨型标题可以使用 `clamp()`，但不要让移动端溢出到不可读。
- 大量留白应服务于结构张力，不要变成普通空白。

## 动效原则

动效是这个方向的核心，不是装饰。

优先级从高到低：

- 鼠标或滚动驱动的线场变形。
- 字符、二进制、斜线组成的流动信息条。
- 章节切换时的大型排版过场。
- 项目悬停时的局部扰动和预览。
- 主题切换时的快速底色替换。

动效约束：

- 运动必须服务“代码正在运行”的气质。
- 避免柔和渐变、玻璃拟态、卡片阴影和常规科技蓝紫色。
- 提供 `prefers-reduced-motion` 降级。
- 移动端保留视觉节奏，但减少复杂鼠标跟随逻辑。

## 图形语言

可复用图形元素：

- 细线透视网格。
- 密集竖线和波形线。
- 二进制数字条。
- 斜线填充块。
- 星形十字标记。
- 裁切后的巨型字母。
- 小型编号，例如 `#3vva-0000/34`。
- 系统状态短句，例如 `RUNNING CREATIVITY PROTOCOLS`。

这些元素应当像界面系统的一部分，而不是随机贴纸。

## 内容语气

文案应简短、直接、有一点技术幽默感。

适合使用：

- “正在构建”
- “可接自由项目”
- “创意开发 / 交互工程 / 动效实现”
- “把设计愿景做成可运行体验”

避免使用：

- 过度营销的价值主张。
- 大段自我介绍。
- 普通模板式按钮文案。
- 和视觉无关的装饰性标语。

## 响应式规则

桌面端：

- 保持强烈横向网格。
- 顶部信息条可以分成多个窄格。
- 巨型标题尽量贴近视口宽度。

移动端：

- 顶部控制条可压缩为两行或图标化。
- 巨型标题允许分行。
- 作品列表保持单列。
- 动态背景降低密度，避免影响阅读。

## 可访问性

- 主题切换按钮必须有明确可访问名称。
- 所有链接保留键盘焦点状态。
- 红黑主题需要检查对比度，正文不要使用过淡透明度。
- Canvas / WebGL 背景不能承载唯一信息。
- 动效必须尊重 `prefers-reduced-motion`。

## 实现建议

- 使用 HTML/CSS 建立边框网格和排版系统。
- 使用 Canvas、WebGL、Three.js 或 SVG 实现线场和字符扰动。
- 使用 CSS 自定义属性统一主题。
- 用 Intersection Observer 驱动章节进入状态。
- 项目数据用结构化数组维护，避免把列表写死在视觉组件里。

## 设计底线

- 不做柔和渐变背景。
- 不做圆角卡片墙。
- 不做常见 SaaS 风格组件。
- 不把作品集做成普通 landing page。
- 不使用大面积蓝紫色科技风。
- 不让动效压过可读性。
- 不让视觉元素看起来随机堆叠。
