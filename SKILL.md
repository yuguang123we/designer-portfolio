# 海报式作品集设计技能

当用户希望作品集接近 `wodniack.dev` 这类实验型创意开发者网站时，使用本技能。

## 使用场景

- 用户想做个人作品集网站。
- 用户偏好强视觉、强动效、强排版。
- 用户喜欢红黑、高对比、代码感或数字海报风格。
- 用户不想要普通模板站、卡片墙或营销落地页。

## 核心目标

把作品集设计成一个可滚动的交互海报系统：

- 用巨型字体建立第一视觉。
- 用细线网格组织版面。
- 用二进制、编号、斜线、字符和坐标制造代码质感。
- 用滚动、悬停和主题切换制造“正在运行”的生命感。
- 用简洁内容承载个人身份、作品和联系方式。

## 执行流程

1. 先确认内容结构
   - 个人身份。
   - 代表能力。
   - 作品列表。
   - 联系方式。
   - 是否需要中英文混排。

2. 建立设计系统
   - 设置红黑 / 纸白黑主题。
   - 定义 display、serif、mono 三类字体。
   - 定义边框、网格、编号和信息条样式。
   - 设定桌面和移动端的标题尺寸。

3. 搭建页面骨架
   - 顶部控制条。
   - 首屏海报。
   - About。
   - Work。
   - Contact。

4. 添加动态层
   - 线场背景。
   - 字符滚动条。
   - 章节过场。
   - 项目悬停预览。
   - 主题切换。

5. 做响应式和可访问性检查
   - 移动端不横向溢出。
   - 按钮和链接可键盘访问。
   - 文字不被动效遮挡。
   - 尊重 `prefers-reduced-motion`。

## 视觉规则

- 优先使用 1px 线条、硬边界和大块色面。
- 标题越大越好，但必须可读。
- 正文应窄栏排版，形成杂志感。
- 作品不要默认做圆角卡片。
- 小字、编号和装饰字符要像系统语言，而不是随机装饰。
- 页面可以夸张，但信息层级必须清楚。

## 推荐设计令牌

```css
:root {
  --color-ink: #160000;
  --color-signal: #f40c3f;
  --color-paper: #fff2ed;
  --font-display: "Bigger Display", "Anton", "Impact", sans-serif;
  --font-serif: "Editorial New", "Noto Serif SC", serif;
  --font-mono: "Fraktion Mono", "IBM Plex Mono", monospace;
  --line: 1px solid currentColor;
}
```

## 常用组件

- `PosterHeader`：顶部控制条。
- `SignalHero`：首屏动态海报。
- `BinaryTicker`：二进制和斜线信息条。
- `WireGrid`：透视线框背景。
- `ProjectIndex`：作品索引列表。
- `ThemeSwitch`：红色 / 纸白主题切换。
- `ContactPoster`：大型文字收尾区。

## 禁止项

- 不使用柔和渐变作为主要背景。
- 不使用大面积玻璃拟态。
- 不使用圆角卡片堆满首页。
- 不写冗长营销文案。
- 不让装饰元素随机漂浮。
- 不让动效影响主要内容阅读。

## 完成标准

一个合格页面应满足：

- 截图时像一张强烈的数字海报。
- 滚动时有章节节奏，而不是普通信息流。
- 作品列表清晰可用。
- 联系方式明确。
- 主题切换自然。
- 桌面和移动端都没有文字重叠、错位或不可点击元素。
