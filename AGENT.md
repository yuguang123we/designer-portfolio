# AGENT 协作说明

本文件用于指导后续在当前项目中工作的 AI 代理或开发协作者。目标是延续 `https://wodniack.dev/` 所启发的实验型作品集方向，并以中文语境完成本地作品集网站的设计与实现。

## 项目目标

构建一个高对比、强排版、带创意开发气质的个人作品集网站。它应该更像一张可滚动的交互海报，而不是普通模板站。

优先感受：

- 第一眼有记忆点。
- 页面结构严谨，像工程图纸。
- 动效和代码质感成为品牌语言。
- 内容简洁，但视觉层次丰富。
- 适合展示设计、影像、交互、前端或创意技术作品。

## 工作原则

- 先理解现有文件，再修改。
- 保持改动聚焦，避免无关重构。
- 如果已有设计资产，优先复用并统一到新视觉系统中。
- 所有新增说明文档使用中文。
- UI 文案使用中文为主，必要时可保留少量英文作为视觉元素。
- 代码实现应服务设计，不要引入复杂但不可见的抽象。

## 设计不可变规则

- 使用强对比色：红黑、纸白黑或黑红。
- 使用细线边框、分栏、编号和字符系统组织页面。
- 标题必须有压倒性存在感。
- 正文必须保持可读，不能为了视觉牺牲阅读。
- 动效应像“系统运行”或“代码生成”，不要像普通装饰动画。
- 作品展示优先使用列表、索引、编号和悬停预览，而不是圆角卡片网格。
- 禁止使用柔和渐变、玻璃拟态、大阴影和模板化英雄区。

## 推荐页面架构

1. 顶部控制条
   - 品牌标识。
   - 导航。
   - 社交链接。
   - 主题切换。
   - 联系入口。

2. 首页海报区
   - 动态线场或字符背景。
   - 二进制 / 斜线 / 编号信息条。
   - 巨型主标题。

3. 关于区
   - 中央正文。
   - 两侧结构线或透视网格。
   - 简洁说明个人能力、经历和合作方式。

4. 作品区
   - 项目索引列表。
   - 每项包含名称、分类、年份、编号、链接状态。
   - 悬停或聚焦时出现预览图、背景扰动或字符变化。

5. 联系区
   - 大型收尾排版。
   - 邮箱或微信等主要联系方式。
   - 不优先做复杂表单。

## 技术建议

- 静态站可以优先使用 Vite、Astro、Next.js 或纯 HTML/CSS/JS。
- 动态线场可用 Canvas、SVG 或 Three.js。
- 页面主题用 CSS 变量控制。
- 项目数据应抽离为结构化数据。
- 需要动效时优先使用 CSS transition、GSAP、Framer Motion 或原生 requestAnimationFrame，按项目栈选择。
- 对移动端和低性能设备提供降级方案。

## 文件约定

- `design.md`：设计系统、视觉语言、布局和动效规则。
- `AGENT.md`：协作代理的项目执行说明。
- `SKILL.md`：可复用的风格执行技能。
- 图片、视频等素材应保留清晰命名，不要随意覆盖原文件。

## 实现检查清单

在交付任何页面改动前，检查：

- 首屏是否有明确视觉冲击。
- 标题是否足够大且没有溢出。
- 移动端是否可读、可滚动、可点击。
- 导航和联系入口是否可用。
- 主题切换是否保持清晰对比。
- 动效是否可关闭或降级。
- 页面是否没有文字重叠和不可读区域。
- 作品列表是否比普通卡片墙更有识别度。

## 交付说明

最终回复用户时，应说明：

- 创建或修改了哪些文件。
- 设计方向的核心是什么。
- 是否进行了本地预览或检查。
- 如未运行预览，应明确说明原因。
